# Lucky_SMS 数据库设计文档

## 1. 数据库概述

Lucky_SMS（学生管理系统）数据库采用MySQL 8.0+，使用utf8mb4字符集和utf8mb4_unicode_ci排序规则，支持完整的Unicode字符集。数据库引擎统一使用InnoDB，提供事务支持和外键约束。

**数据库特性：**
- 字符集：utf8mb4
- 排序规则：utf8mb4_unicode_ci
- 存储引擎：InnoDB
- 支持事务和外键约束
- 完整的索引优化

## 2. 数据库表结构

### 2.1 基础字典表

#### 2.1.1 用户表 (users)
存储系统所有用户的基础信息。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| user_id | INT | PRIMARY KEY, AUTO_INCREMENT | 用户ID（主键） |
| username | VARCHAR(50) | UNIQUE, NOT NULL | 用户名（全局唯一） |
| password | VARCHAR(100) | NOT NULL | 用户密码 |
| phone | VARCHAR(20) | - | 手机号码 |
| email | VARCHAR(100) | - | 电子邮箱 |
| avatar | VARCHAR(200) | - | 头像URL |
| real_name | VARCHAR(50) | - | 真实姓名 |
| gender_id | INT | FOREIGN KEY | 性别ID |
| birthday | DATE | - | 出生日期 |
| address | VARCHAR(200) | - | 地址 |
| user_type_id | INT | FOREIGN KEY, NOT NULL | 用户类型ID |
| status_id | INT | FOREIGN KEY, DEFAULT 1 | 状态ID |
| created_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

| updated_by | INT | FOREIGN KEY | 更新人ID |

  - sort_order: 排序顺序

- **审核状态表 (review_statuses)**：定义审核流程状态
  - review_status_id: 审核状态ID（主键）
  - status_name: 状态名称
  - description: 状态描述
  - is_active: 是否激活

- **性别表 (genders)**：定义性别选项
  - gender_id: 性别ID（主键）
  - gender_code: 性别代码
  - gender_name: 性别名称

- **课程类型表 (course_types)**：定义课程类型
  - course_type_id: 课程类型ID（主键）
  - type_code: 类型代码
  - type_name: 类型名称
  - description: 类型描述
  - is_active: 是否激活
  - sort_order: 排序顺序

- **学期类型表 (semester_types)**：定义学期类型
  - semester_type_id: 学期类型ID（主键）
  - type_code: 类型代码
  - type_name: 类型名称
  - description: 类型描述

- **选课类型表 (selection_types)**：定义选课类型
  - selection_type_id: 选课类型ID（主键）
  - type_code: 类型代码
  - type_name: 类型名称
  - description: 类型描述

- **成绩等级表 (grade_levels)**：定义成绩等级
  - grade_level_id: 成绩等级ID（主键）
  - level_code: 等级代码
  - level_name: 等级名称
  - min_score: 最低分数
  - max_score: 最高分数
  - gpa_points: GPA点数

- **考核方式表 (assessment_methods)**：定义考核方式
  - assessment_method_id: 考核方式ID（主键）
  - method_code: 方式代码
  - method_name: 方式名称
  - description: 方式描述
